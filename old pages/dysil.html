Dysil for Dynamic Signal

The overarching objective for this project is to allow the Dynamic Signal platform to be extensively customized based on the needs of each client, both on the UI as well as the backend. I was brought in to solve how we would incorporate these custom built features into the platform and how our end users would interact with them.

Timeline:
Feb - Apr 2019
My Role:
Product Designer
Function:
Product & UX Design
Category:
Android, iOS, Web
The Situation

As an enterprise platform, Dynamic Signal has always dealt with custom feature requests for the numerous comms use cases that our customers are met with -- Paystubs, Work Schedule, Weather Widgets, Stock Prices...the list goes on and on.

The engineering team planned to build an engine (Dysil) that would provide the framework & tools to support these use cases without having to build each from scratch. Dysil gives customers the ability to write and run scripts that use Microsoft's Adaptive Cards to render for iOS, Android, and HTML5.

As the designer brought onto this massive engineering project, I was tasked with detailing how our end user (the employee) would interact with these custom built cards in cohesion with our existing mobile and web platform.

The Result

I'm incredibly proud of the final product which resulted in a near 65% adoption across all of Dynamic Signal's customers. There was great value in designing this feature across multiple platforms and delivery methods. Quick Polls is one of the few pieces of content in the platform that could be posted in a news feed, broadcasted out in the form of a notification, sent through email, or shared via direct messaging.

From a design perspective, this was an incredibly rewarding experience where I was able to own the complete end to end design experience from the initial problem statement to ideation to user research to crafting user flows to designing the UI across platforms to final prototypes.
